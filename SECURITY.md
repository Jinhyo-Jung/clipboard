# SECURITY

이 문서는 민감정보 및 보안 관련 저장소 운영 규칙을 정의한다.

## 민감정보 커밋 금지
- API 키, 액세스 토큰, 비밀번호, 개인정보, 내부 IP 등 민감정보는 Git에 커밋하지 않는다.

## 비밀정보 관리 방식
- 환경변수 사용을 우선한다.
- 로컬 파일이 필요한 경우 비추적 파일로 관리한다.
  - 예: `.env`, `secrets.json`, `config.local.json`, `*.key`

## .gitignore 정책
- 민감정보 파일/패턴은 반드시 `.gitignore`에 등록한다.
- 기본 차단 항목:
  - `.env`
  - `.env.*`
  - `secrets.json`
  - `config.local.json`
  - `*.pem`
  - `*.key`
  - `*.p12`
  - `*.mobileprovision`

## 사고 대응
- 민감정보가 커밋되었다면 즉시 키 폐기/재발급을 수행한다.
- 유출 커밋 제거가 필요한 경우, 영향 범위를 확인한 뒤 히스토리 정리 절차를 진행한다.
