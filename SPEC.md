# 맥용 단축키 중심 클립보드 앱 스펙 (초안)

## 목적
- `CopyClip - Clipboard History`(App Store)와 Windows의 Win+C/V 경험을 참고하여, macOS에서 마우스 사용 없이 전역 단축키로 모든 클립보드 흐름을 제어하는 앱의 방향성을 정리한다.

## 제품 비전
- 전역 단축키 한 번으로 히스토리 패널을 열고, 키보드만으로 항목 탐색/붙여넣기를 즉시 수행한다.
- 무마우스 사용성, 경량 백그라운드 동작, 최소한의 UI 요소에 집중한다.
- CopyClip의 핵심 기능(히스토리, 검색, 저장 제한, 삭제)을 그대로 수용하되, 추가적인 접근성 흐름을 제공한다.

## 사용자 흐름
1. 사용자가 문서/브라우저 등에서 텍스트를 복사한다.
2. 전역 단축키(예: `⌃⌥V`)를 누르면 히스토리 패널이 즉시 레이어로 나타난다.
3. 방향키/숫자키/검색으로 원하는 항목을 선택하고 `Enter`를 누르면 현재 포커스 앱에 붙여넣는다.
4. 패널은 자동 닫히고, 사용자는 작업을 재개한다. 필요한 경우 `Esc`로 수동 종료 가능.

## UX/기능 원칙
- **즉시성**: 열기 지연 150ms 이내 / 단축키 누름 즉시 표시.
- **무마우스**: 모든 동작에 키보드 경로 제공 (선택, 삭제, 설정). 마우스는 옵션.
- **단순 메뉴**: 기본 화면은 히스토리 목록 + 검색바 + 상태 표시.
- **예측 가능한 동작**: 선택 후 `Enter`는 항상 붙여넣기, `⌘⌫`는 삭제 등 일관성 유지.

## MVP 기능
- 클립보드 히스토리 저장
  - 시스템 클립보드 변경 감지 (텍스트 기준)
  - 중복 항목은 순위 재조정 (최근/빈도 기반 옵션 추가 고려)
- 전역 단축키 패널
  - `⌃⌥V`(초기 추천, 사용자 지정 가능)
  - 패널은 화면 중앙 또는 마지막 사용 위치
- 키보드 탐색/붙여넣기
  - `↑/↓`, `⌘1..9` 키 입력, 또는 검색 타이핑 후 `Enter`
  - `Esc` 닫기, `⌘↩` 빠른 붙여넣기(옵션)
- 검색 필터
  - 패널 오픈 직후 타이핑 시 실시간 필터
  - 대소문자 구분/정규식은 Beta
- 설정/관리
  - 시작 시 자동 실행, 최대 저장 개수 설정(50/100/200)
  - 개별 삭제(`⌘⌫`), 전체 삭제(`⇧⌘⌫`)
  - 클립 기록 일시 중지 토글
- 메뉴바 상주
  - 메뉴바 아이콘으로 상태 확인 + 단축키 안내

## CopyClip 참고 기능 매핑
- 히스토리 저장 및 목록 표시 → MVP에서 동일 구현
- 검색 바 필터 → MVP 검색 필터 포함
- 개별/전체 삭제 → 설정 기능에 포함
- 저장 개수(remember count) 설정 → 기본 설정 포함
- 자동 시작 → 설정에 포함
- 앱 예외 목록 / 핀 항목 / 클립 편집 / 붙여넣기 옵션 등은 Beta 이후
- 이미지/파일 클립, 클라우드 동기화 같은 고급 기능은 초기 범위 제외

## 단축키 초안
- 패널 열기: `⌃⌥V`
- 선택 항목 붙여넣기: `Enter`
- 단축키 탐색: `↑/↓`, `⌘1..9`
- 검색 시작: 오픈 상태에서 바로 타이핑
- 삭제: `⌘⌫` (선택 항목 삭제), `⇧⌘⌫` (전체 삭제)
- 패널 닫기: `Esc`
- 실행 중단/빠른 복사: `⌘C`는 시스템 복사 유지

## 요구사항
- 전역 단축키 충돌 감지 및 재정의 옵션
- 붙여넣기는 현재 포커스 앱에 수행
- 로컬 저장 기본, 민감 앱 제외 옵션은 Beta
- 리소스: 메뉴바 앱으로 백그라운드 작동, 메모리/CPU 적게 사용
- 접근성: 권한 안내 및 수동 붙여넣기 대체

## 리스크/대응
- 접근성 권한 없이 붙여넣기 실패 → Onboarding에서 권한 안내 및 수동 복사 안내
- 비정상 단축키 충돌 → 기본값 충돌 감지 후 추천 키 제안
- 클립보드 누락 → 이벤트 버퍼링/예외 로그

## 출시 단계
1. Alpha: 텍스트 히스토리 + 단축키 패널 + `Enter` 붙여넣기
2. Beta: 검색 고도화, 설정(개수 제한/삭제), 자동 실행, 접근성 안내
3. v1.0: 앱 예외 목록, 핀, 클립 편집, 사용자 단축키 저장

## 프로젝트 운영 규칙 v2
- 변경 기록 파일은 `docs/CHANGE_LOG.md`로 고정한다. (`CHANEGE_LOG.md` 금지)
- 답변, 문서, 코드 주석은 한글을 기본으로 작성한다.
- 커밋/푸시는 기능 단위로 수행하고 작업 종료 시 원격 저장소에 반영한다.
- 앱의 기술/기능/제약사항은 본 `SPEC.md`를 단일 기준 문서(SSOT)로 유지한다.
- 프로젝트 대상은 macOS 사용자용 앱으로 한정한다.

## 브랜치/릴리스 전략
- `main`: 안정 브랜치
- `release/*`: 배포 준비 브랜치
- `codex/*`: 기능 작업 브랜치
- 병합 전 필수 조건: 코드 변경 + `SPEC.md` 반영 + `docs/CHANGE_LOG.md` 반영

## 보안 및 비밀정보 관리
- API 키/토큰/비밀번호/개인정보는 절대 Git에 커밋하지 않는다.
- 비밀정보는 환경변수 또는 비추적 파일(`.env`, `secrets.json`, `config.local.json`, `*.key`)로 관리한다.
- 비밀정보 파일/패턴은 반드시 `.gitignore`에 등록한다.

## 변경 로그 작성 규칙
- `docs/CHANGE_LOG.md`의 모든 항목은 날짜와 시간(로컬 타임존)을 포함한다.
- 변경 로그는 최신 항목이 항상 상단에 오도록 최신순으로 정렬한다.
- 항목 템플릿은 아래 형식을 사용한다.
  - 일시
  - 유형(`feat`, `fix`, `docs`, `refactor`, `chore`)
  - 요약
  - 변경 파일
  - 검증 결과
  - 리스크/후속 작업
