# CONTRIBUTING

이 문서는 협업 시 브랜치, 커밋, 병합 기준을 정의한다.

## 브랜치 전략
- `main`: 안정 브랜치
- `release/*`: 배포 준비 브랜치
- `codex/*`: 기능 작업 브랜치

## 작업 흐름
1. 기능 작업은 `codex/*` 브랜치에서 진행한다.
2. 기능 단위로 커밋한다.
3. 배포 준비 시 `release/*`로 병합한다.
4. 검증 완료 후 `main`에 반영한다.

## 병합 전 필수 체크
- 코드 변경이 있으면 `SPEC.md` 반영 여부 확인
- 모든 작업에 대해 `docs/CHANGE_LOG.md` 기록 완료
- 변경 로그가 최신순(최신 항목 상단)인지 확인
- 변경 로그 항목에 날짜/시간이 포함되었는지 확인

## 커밋 메시지 권장 형식
- `feat: ...`
- `fix: ...`
- `docs: ...`
- `refactor: ...`
- `chore: ...`

## 원격 저장소
- 기본 원격: `git@github.com:Jinhyo-Jung/clipboard.git`
- 작업 종료 시 `push`로 원격 반영
